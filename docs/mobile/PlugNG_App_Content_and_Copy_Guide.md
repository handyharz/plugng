# PlugNG App Content and Copy Guide

## Purpose
Define the writing style and reusable copy patterns for the PlugNG app so the product feels premium, clear, and trustworthy.

## Brand Voice
PlugNG should sound:

- confident
- sharp
- premium
- direct
- trustworthy

It should not sound:

- generic
- overly playful
- verbose
- corporate and cold

## Writing Principles

### 1. Be Clear First
Users should understand what to do immediately.

### 2. Sound Premium, Not Pretentious
Short lines can feel elevated without becoming vague.

### 3. Reassure at High-Risk Moments
Especially during:

- wallet funding
- checkout
- order tracking
- support

### 4. Keep Labels Short
Buttons and section headers should be fast to scan.

## Copy Rules for Buttons

- use verb-led labels
- avoid vague labels like `Continue` when more specific text fits
- keep button text short enough to scan instantly

Preferred:

- `Fund Wallet`
- `Place Order`
- `Track Order`

Less preferred:

- `Proceed`
- `Submit`
- `Done`

## Notification Copy Direction

Examples:

- `Your order has been confirmed`
- `Your wallet top-up was successful`
- `An item in your wishlist is on sale`
- `Your order is out for delivery`

## Error Message Rules

- explain what happened simply
- tell the user what to do next
- avoid raw technical language

Formula:

- what happened
- what to do now

Example:

- `Unable to load your wallet balance. Please try again.`
