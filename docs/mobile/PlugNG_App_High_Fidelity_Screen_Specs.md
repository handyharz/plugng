# PlugNG App High-Fidelity Screen Specs

## Purpose
Translate the wireframe plans into high-fidelity screen direction for design production and engineering handoff.

This document focuses on:

- visual hierarchy
- content priority
- exact section intent
- recommended copy direction
- design notes for high-fidelity mockups

## Screens Covered

1. Home
2. Search
3. Product Detail
4. Cart
5. Checkout

## Shared High-Fidelity Rules

### Visual Tone

- premium dark UI
- strong contrast
- clean glass surfaces
- product-first layouts
- limited use of secondary accent colors

### Copy Tone

- direct
- premium
- trust-first
- concise

### High-Fidelity Priority
Each screen comp should answer:

- what is the first thing the user notices
- what the user should do next
- what makes the experience feel trustworthy

## 1. Home High-Fidelity Spec

## Primary Objective
Make the app feel immediately premium and useful while encouraging fast search and browsing.

## Above-the-Fold Structure

1. compact brand header
2. dominant search entry
3. category shortcuts
4. wallet promo or balance card
5. first product row begins within or just below first viewport

## Visual Priority

### First visual focus

- search bar

### Second visual focus

- wallet card

### Third visual focus

- first product row

## Header Spec

### Layout

- left: PlugNG brand mark or compact wordmark
- right: notifications and cart

### Styling

- transparent or near-transparent over dark canvas
- subtle blur only if helpful
- icons should use muted text until active

## Search Entry Spec

### Role
The search entry is the main hero on mobile.

### Styling

- full-width rounded input surface
- dark glass fill
- blue focus ring or edge glow
- leading search icon
- concise placeholder

### Placeholder Options

- `Search accessories, brands, or device models`
- `Search cases, chargers, audio gear`

### Notes

- do not put too much decorative content above this
- this should feel like the center of the Home experience

## Category Shortcuts Spec

### Format

- horizontal pill row or icon-led chips

### Recommended Categories

- Cases
- Chargers
- Audio
- Screen Protectors
- Power Banks
- Adapters

### Styling Notes

- each chip should feel tactile
- active or pressed state should use blue or elevated glass styling

## Wallet Card Spec

### Role
Acts as the app's strategic monetization and retention card.

### Variants

- signed-in with balance
- signed-in without balance
- guest view with wallet teaser removed or softened

### Content Direction

#### Balance Variant

- label: `Wallet Balance`
- balance value
- supporting line: `Faster checkout. Instant payment confirmation.`
- CTA: `Top Up`

#### Promo Variant

- headline: `Top up and unlock faster checkout`
- supporting line: `Use wallet for smoother purchases and special offers.`
- CTA: `Fund Wallet`

### Styling Notes

- use a stronger glow than standard cards
- do not let it look like a financial dashboard
- should feel like a premium utility card

## Featured Products Section Spec

### Role
Primary merchandising zone.

### Section Header

- title: `Featured`
- optional action: `View All`

### Card Style

- larger visual cards
- stronger imagery
- price clearly visible
- optional sale/wallet badge

### Notes

- first row should feel curated, not crowded
- use fewer cards with higher quality visual treatment

## Trending / Deals Spec

### Role
Secondary browse layer.

### Section Titles
Examples:

- `Trending Now`
- `Deals Worth Grabbing`

### Card Style

- slightly smaller than featured cards
- supports faster browsing

## Continue Shopping Spec

### Role
Retention and continuity.

### Show Only If

- recent browsing exists
- cart has activity
- wishlist has relevant items

### Empty Replacement

- `Recommended for You`

## Trust Strip Spec

### Role
Add low-friction reassurance without becoming a giant trust banner.

### Recommended Items

- Authentic Gear
- Secure Pay
- Fast Delivery

### Styling Notes

- compact, muted, high readability
- small icons optional

## Home Recommended Copy Samples

- `Find your next upgrade`
- `Search accessories, brands, or device models`
- `Wallet makes checkout faster`
- `Selected for your setup`

## 2. Search High-Fidelity Spec

## Primary Objective
Make search feel premium, focused, and fast enough to become the app's power feature.

## Screen Structure

1. top bar with back/cancel
2. expanded search input
3. recent searches
4. trending searches
5. category suggestions
6. brand suggestions

## Search Input Spec

### Styling

- large input
- dark surface with strong border definition
- bright cursor
- clear button on the right

### Placeholder

- `Search products, brands, or phone models`

### Interaction Notes

- keyboard appears on open
- suggestions should feel instant

## Recent Search Block

### Role
Accelerate repeat discovery.

### Format

- compact chips
- horizontally wrapping or scrollable

### Copy Tone

- no heading overload
- simple label like `Recent`

## Trending Search Block

### Role
Create discovery energy and guide indecisive users.

### Recommended Query Style

- `iPhone 15 case`
- `30W fast charger`
- `Type-C adapter`
- `Oraimo earbuds`

### Styling

- compact cards, not plain text rows

## Suggested Categories and Brands

### Role
Offer low-friction browse entry.

### Styling Notes

- categories can be icon-led
- brands can be compact pills
- keep background calmer than Home

## Search Recommended Copy Samples

- `Search`
- `Recent`
- `Trending`
- `Shop by category`
- `Popular brands`

## 3. Product Detail High-Fidelity Spec

## Primary Objective
Turn interest into purchase confidence.

## Top Section Hierarchy

1. media gallery
2. title and price
3. compatibility
4. trust and delivery
5. sticky purchase bar

## Gallery Spec

### Role
Show the product as premium and desirable.

### Styling Notes

- generous media height
- minimal chrome around images
- subtle pagination dots
- support fullscreen later if needed

## Title and Price Spec

### Layout

- title on one or two lines max
- price below with strong contrast
- compare-at price smaller and muted
- sale or wallet badge nearby, not stacked excessively

### Price Styling

- bold
- large
- easy to scan

## Compatibility Card Spec

### Role
One of the most important conversion blocks.

### Content Examples

- `Works with iPhone 15 Pro`
- `Designed for MagSafe`
- `Compatible with USB-C devices`

### Styling Notes

- stronger than a generic card
- blue or brand emphasis border
- icon optional but should not overpower text

## Trust Row Spec

### Role
Provide quick reassurance before the CTA.

### Items

- Authentic Product
- 7-Day Returns
- Secure Checkout

### Styling Notes

- horizontal mini cards or badge row
- readable, not tiny

## Delivery Info Spec

### Role
Clarify speed and reduce hesitation.

### Content Examples

- `Estimated delivery: 2-4 business days`
- `Ships nationwide`

### Notes

- delivery should feel like useful information, not promo fluff

## Sticky Purchase Bar Spec

### Layout

- left side optional price recap or wishlist icon
- primary CTA: `Add to Cart`
- secondary CTA: `Buy Now`

### Styling Notes

- should be visually separated from page content
- use strong blue button for primary action
- secondary should still feel available, not hidden

## Product Detail Recommended Copy Samples

- `Authentic gear only`
- `Secure payment via trusted checkout`
- `Fast delivery across Nigeria`
- `Works with your device`

## 4. Cart High-Fidelity Spec

## Primary Objective
Make the user feel one confident step away from checkout.

## Screen Structure

1. header
2. cart items
3. coupon/promo
4. wallet nudge
5. summary
6. sticky checkout CTA

## Cart Item Spec

### Layout

- image left
- product info center
- price and quantity controls right or below

### Visual Rules

- item cards should be clear and stable
- avoid too much decorative styling here

## Coupon Module Spec

### Role
Support promo behavior without dominating the cart.

### Styling

- utility-first
- compact
- visible applied success state

## Wallet Nudge Spec

### Role
Encourage wallet adoption when relevant.

### Content Examples

- `Use wallet for faster checkout`
- `Top up to unlock smoother payment`

### Rules

- only show if useful
- keep less visually loud than the final total

## Summary Spec

### Content

- subtotal
- discounts
- delivery estimate
- total

### Visual Rules

- total should be the strongest line in the summary block

## Sticky CTA Spec

### Button Copy

- `Proceed to Checkout`

### Notes

- sticky CTA should remain visible
- disabled state must still be readable

## 5. Checkout High-Fidelity Spec

## Primary Objective
Make the transaction feel low-friction and safe.

## Design Tone

- cleaner than Home
- quieter than Product Detail
- less merchandising
- more trust and clarity

## Checkout Stepper Spec

### Recommended Labels

1. Address
2. Payment
3. Review

### Styling

- compact top progress indicator
- blue highlight for active step
- muted inactive steps

## Address Step Spec

### Priority

- saved addresses first
- form second

### Address Card Styling

- clear selected state
- strong legibility
- not visually overloaded

### CTA Copy

- `Continue to Payment`

## Payment Step Spec

### Priority

- recommended method first
- strong trust cue second
- continue CTA third

### Wallet Card Content

- current balance
- amount left after purchase if sufficient
- shortfall if insufficient

### Payment Method Labels

- `Wallet`
- `Card`
- `Bank Transfer`

### CTA Copy

- `Continue to Review`

## Review Step Spec

### Purpose
Calm the user before final confirmation.

### Content Blocks

- item summary
- address summary
- payment summary
- cost breakdown

### CTA Copy

- `Place Order`

### Security Copy

- `Your payment is secured before confirmation`

## Success Screen Spec

### Structure

- success icon or motion
- headline
- order number
- `Track Order`
- `Continue Shopping`

### Suggested Copy

- `Order Confirmed`
- `Your Plug is secured`

## High-Fidelity Design Checklist

For every screen comp, confirm:

1. one element has clear primary emphasis
2. CTA hierarchy is obvious
3. trust cues are near decisions
4. dark surfaces still preserve readability
5. accent colors are not overused
6. product imagery has enough room
7. empty/loading/error states are designed too
